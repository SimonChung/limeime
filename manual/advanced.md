# 進階使用

已經能穩定輸入並完成資料庫備份後，再處理自製輸入法、資料庫進階管理、學習系統與自訂鍵盤佈局。進階操作可能覆蓋資料或改變候選字結果，動手前請先備份完整資料庫。

## 自製輸入法

如果內建下載清單沒有你的碼表，或你想維護自己的字根、候選字與符號規則，可以自製輸入法。

### 碼表檔案格式

LIME 可匯入 `.cin`、`.lime` 與 `.limedb`，使用者選擇檔案格式時，應先確認自己要匯入文字碼表、單一輸入法資料，還是關聯字庫資料。

`.cin` 是傳統中文輸入法社群常見的文字碼表格式，檔案用文字記錄字根與候選字。使用者從舊輸入法、輸入法社群或個人碼表整理工具取得碼表時，常會看到 `.cin` 檔。

`.lime` 是 LIME 的文字碼表格式，檔案同樣用文字記錄字根與候選字，也可保存 LIME 使用的輸入法名稱、版本與結束鍵等資訊。使用者需要用文字編輯器檢查、修改或保存碼表時，可以使用 `.lime`。

`.limedb` 是 LIME 的單一輸入法資料表備份與還原格式。使用者想把某個輸入法分享給其他使用者，或只想把單一輸入法從一台裝置搬到另一台裝置時，請使用 `.limedb`。檔案可保存該輸入法的字根、候選字、分數與輸入法資訊。關聯字庫也使用 `.limedb`，但關聯字庫檔案必須從「關聯字庫」分組匯入。

需要確認 `.cin` 與 `.lime` 的欄位、編碼與範例時，請參考 [CIN_LIME_SPEC.md](../docs/CIN_LIME_SPEC.md)。

匯入自製碼表前請先確認：

- 檔案格式正確，例如 `.cin`、`.lime`、`.limedb`。
- 編碼與字根區段正確。
- 已備份目前資料庫。
- 知道匯入後要測試哪些字根與候選字。

自製碼表完成後，請到 [輸入法管理](ime-management.md) 匯入並測試。

## 資料庫進階管理

`lime.db` 是 LIME 的主要資料庫，通常包含輸入法碼表、字根資料、候選字、關聯字詞、學習資料與部分設定。

跨裝置同步最可靠的方式是手動備份與還原：在舊裝置備份完整資料庫，再到新裝置還原。若只重新下載碼表，通常無法保留完整個人資料。

若輸入法列表空白、候選字消失或設定 App 異常，請先備份目前狀態，再嘗試還原舊備份或還原預設資料庫。

## 學習系統

LIME 會依使用者選字與輸入習慣調整候選字排序，學習資料通常保存在資料庫中。

候選字排序可能受到以下因素影響：

- 碼表原始順序。
- 使用者學習資料。
- 關聯字詞。
- 智慧組詞設定。
- 最近是否還原、重設或重新下載碼表。

若要保留學習資料，換機時請使用完整資料庫備份還原。

## 自訂鍵盤佈局

LIME 的可見鍵盤佈局和輸入法碼表不是同一件事，修改鍵盤佈局前，請先確認你要改的是按鍵位置、按鍵輸出、長按符號，還是碼表中的字根資料。

Android 鍵盤佈局和 XML 資源相關，iOS/iPad 鍵盤佈局使用 JSON 版面，iPad 另有五列鍵盤、副鍵符號與分割鍵盤差異。

修改後至少測試：

1. 中文輸入。
2. 英文輸入。
3. 符號頁。
4. Email、密碼、URL、搜尋、電話與數字欄位。
5. iPad 橫向與直向。
6. 長按迷你鍵盤與副鍵符號。

iPad 尺寸分級目前未實作，不要把它當成可用的自訂佈局能力。
