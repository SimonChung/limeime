## B. Data Safety form answers (Section 8)

Fill the Play Console **Data Safety** wizard with these answers. The honest
position for LimeIME is **"No data collected, no data shared."**

### Top-level

| Question | Answer |
| --- | --- |
| Does your app collect or share any of the required user data types? | **No** |
| Is all collected data encrypted in transit? | N/A (nothing collected/transmitted) |
| Do you provide a way for users to request data deletion? | Data is on-device; users delete via Clear data / Uninstall |

> Rationale: LimeIME stores typing/learning data **only on the device** and never
> transmits it. On-device-only storage is **not** "collection" under Play's Data
> Safety definitions. The network permission is used solely to download
> input-method tables the user requests — that is not collection of user data.

### If Play flags the microphone or you prefer to over-declare

If the review asks about `RECORD_AUDIO`, declare:

| Field | Answer |
| --- | --- |
| Data type | Audio → Voice or sound recordings |
| Collected? | **No** — audio is processed by the system speech service, not stored or sent by LimeIME |
| Shared? | No |
| Purpose | App functionality (voice typing) |

### Permissions declaration (if prompted)

| Permission | Declared use |
| --- | --- |
| `RECORD_AUDIO` | Optional in-keyboard voice typing; recognition delegated to the system speech service. Not used to record/store/transmit audio. |
| `INTERNET` / `ACCESS_NETWORK_STATE` | Download user-selected input-method tables and check connectivity. |
| `POST_NOTIFICATIONS` | Status/progress notifications. |
| `VIBRATE` | Optional key-press haptics. |

## F. Still requires a human decision / action (not draftable here)

- **Package identity — DECIDED: brand-new listing under `org.limeime`.**
  The old `net.toload.main.hd` listing (500k installs / 4.3★) **cannot be reused**
  — its Play developer account belongs to a now-inactive team member and is not
  under this team's control. Implications to accept:
  - This is a **fresh listing**: no inherited installs, reviews, or ratings; the
    rating/review count starts at zero.
  - Existing `net.toload.main.hd` users will **not** receive this as an update —
    it is a separate app they must find and install anew.
  - The Play listing id is **`org.limeime`** — **version- and ownership-neutral**.
    Chosen over the original `net.toload.*` prefix (named by the now-inactive
    original developer, meaning unknown) and over a year-stamped `hd2026`. ⚠️ This
    id is **permanent once published** — confirm it before the first upload, because
    it can never be changed afterward. *(Note: current beta builds still ship as
    `net.toload.main.hd2026`; `applicationId` is switched to `org.limeime` at Play
    onboarding — edit [build.gradle](../LimeStudio/app/build.gradle#L34) + the
    [test assertion](../LimeStudio/app/src/androidTest/java/net/toload/main/hd/ApplicationTest.java#L74) then.)*
  - **The Java code is NOT moved** in either phase. The `applicationId` (the
    Play/app identity, whatever it is) and the Java `namespace`
    (`net.toload.main.hd`, where classes live) are independent by design. Classes stay under `net.toload.main.hd`; the
    manifest's relative names (`.LIMEService`, `.LIMEBackupAgent`, `.ui.LIMESettings`)
    resolve against the unchanged namespace. Renaming the Java package would be a
    large, risky refactor with **no user-visible benefit** — deliberately avoided.
  - In the store description, frame LIME IME 6 as the **continuation/successor** of
    the original LIME IME (the draft already does this) so returning users recognise
    it, even though the listing itself is new.
- **Developer/store display name**: cannot reuse the historic `Android.toload.net`
  account. Use this team's account; suggested display name
  **`The LimeIME Open Source Project`** (or your own).
- **Signing & AAB** (GOOGLE_PLAY.md Sections 2–3): create the upload keystore and
  wire a `signingConfig`; build a signed `.aab`. Left out per scope choice.
- **Distribution countries** (Section 12): choose target regions.
- **Publish the privacy policy** to GitHub Pages and confirm the live URL before
  entering it in Play Console.
- **Generate the 512 icon + feature graphic** from a hi-res master (design task).
